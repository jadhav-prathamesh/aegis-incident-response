Evidence: What We've Built and Tested
========================================

This document tracks what evidence exists to prove our platform works.

What's Fully Built (Source Code)
--------------------------------
All 5 AI agents (Orchestrator, Planner, Executor, Observer, Validator) are fully implemented in src/agents/ with typed LangGraph state machines. Each agent has:
- Dedicated state class with agent-specific fields
- Custom system prompts and tool sets
- Process/tools/evaluate lifecycle with iteration limits
- Confidence scoring and error handling

API: 23 endpoints across 6 route modules (health, incidents, agents, approvals, monitoring, validation) built with FastAPI.

Core Services (14 modules):
- Config: 11 nested settings groups with env override
- Database: PostgreSQL async engine with connection pooling
- Vector DB: ChromaDB + pgvector dual support
- Knowledge Base: Hybrid vector + keyword search with 5 seed runbooks
- Action Dispatcher: 15 action types with individual handlers
- Approval Workflow: Request/approve/reject lifecycle with auto-approve in dev
- Monitoring: Health checks, Prometheus queries, Alertmanager, log analysis
- Validation: 6 check types (health, metrics, synthetic, alerts, compliance, baselines)
- Incident Store: In-memory with PostgreSQL engine ready
- Similar Incidents: Embedding-based similarity search with brute-force fallback
- Embeddings: Provider API with deterministic local fallback

Dashboard: Streamlit with 4 pages (Overview, Incidents, Approvals, Create Incident)
Deployment: Docker Compose with 7 services, Dockerfile, health checks
