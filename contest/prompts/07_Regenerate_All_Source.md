# Regeneration Prompt: All Contest Source Files

## Objective

Batch-regenerate all 15 contest source files from the codebase after updates.

## Inputs

All files in `contest/source/` and all code files in `src/`.

## Execution Order

1. `Contest_Project_Summary.md` — Top-level overview
2. `Contest_Problem_Statement.md` — Problem definition
3. `Contest_Customer_Challenges.md` — Challenge mapping
4. `Contest_AI_Solution.md` — AI/ML architecture
5. `Contest_Solution_Architecture.md` — System architecture
6. `Contest_Agentic_Workflow.md` — Agent pipeline
7. `Contest_Business_Value.md` — Value proposition
8. `Contest_Implementation_Timeline.md` — Development phases
9. `Contest_Deployment.md` — Deployment documentation
10. `Contest_Future_Roadmap.md` — Planned features
11. `Contest_Assumptions.md` — Assumptions register
12. `Contest_Evidence.md` — Evidence and traceability
13. `Contest_Risk_Register.md` — Risk documentation
14. `Contest_Demo_Script.md` — Demo walkthrough
15. `Contest_QA.md` — Quality assurance

## Common Verification

For each file:
- [ ] Status labels use permitted vocabulary only
- [ ] Every claim is traceable to source code
- [ ] No fabricated data, metrics, or customer references
- [ ] Cross-references to other files are valid
- [ ] Business value projections are labelled "Estimated"
